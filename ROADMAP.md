  - **ส่งภาพผ่าน LINE**: เดิมตัดสินใจไม่ส่งเพราะ LINE Messaging API ต้องการ URL ภาพที่ดึงได้แบบไม่ต้อง auth header — ผู้ใช้ยืนยันให้ทำโดยใช้ **URL ชั่วคราวแบบมี token สุ่มเดาไม่ได้** (แนะนำจาก 3 ตัวเลือกที่เสนอ) แทนการเปิดเป็น URL สาธารณะถาวร: เพิ่ม `backend/src/utils/imageToken.ts` เซ็น token ด้วย HMAC-SHA256 (ใช้ secret เดียวกับ JWT) ฝัง path ของภาพ + เวลาหมดอายุ (~15 นาที) ไว้ในตัว token เอง ไม่ต้องมีตารางในฐานข้อมูล/งาน cleanup; เพิ่ม endpoint สาธารณะ `GET /api/notifications/image/:token` ที่ verify signature+อายุ token ก่อนเสิร์ฟไฟล์ (token ปลอมหรือหมดอายุ → 404 เสมอ ทดสอบแล้ว)
  - เพิ่มตัวแปร `.env` ใหม่ `PUBLIC_BASE_URL` (โดเมนสาธารณะจริงของระบบรวม `/khd-faceto` เช่น `https://attendance.example.go.th/khd-faceto`) — ถ้าไม่ได้ตั้งค่าไว้ ระบบจะส่งแจ้งเตือน LINE แบบข้อความอย่างเดียวเหมือนเดิมโดยไม่ error (เพราะ LINE server ดึง URL เองโดยตรง เข้าถึง IP ในเครือข่าย LAN หรือใบรับรอง HTTPS ที่ออกเองสำหรับกล้องไม่ได้)
  - **ทดสอบแล้ว**: ย่อขนาดภาพทำงานถูกต้อง (ดูด้านบน); endpoint `/api/notifications/image/:token` คืนภาพถูกต้องเมื่อ token ถูกต้อง (200) และคืน 404 เมื่อ token ปลอม/ไม่มีลายเซ็นถูกต้อง/พยายาม path traversal (ทดสอบแยกทั้งกรณี garbage token และ token ที่ปลอม payload เป็น `../../../etc/passwd` แต่ signature ไม่ตรง) — ยังไม่ได้ทดสอบส่งจริงผ่าน LINE เพราะเครื่อง dev ไม่มีโดเมนสาธารณะให้ตั้ง `PUBLIC_BASE_URL` จริง (ต้องตั้งค่าและทดสอบจากเซิร์ฟเวอร์จริงของหน่วยงาน)
